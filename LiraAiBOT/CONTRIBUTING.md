## 🚀 Как внести вклад

### 1. Fork репозитория

Нажмите кнопку **Fork** в правом верхнем углу GitHub.

### 2. Клонируйте репозиторий

```bash
git clone https://github.com/YOUR_USERNAME/LiraAiBOT.git
cd LiraAiBOT
```

### 3. Создайте ветку

```bash
git checkout -b feature/your-feature-name
```

**Примеры имён веток**:
- `feature/payment-integration`
- `bugfix/telegram-auth`
- `docs/update-readme`

### 4. Внесите изменения

Следуйте стандартам кода ниже.

### 5. Протестируйте

```bash
# Запустите тесты
python -m pytest tests/

# Проверьте стиль
flake8 backend/

# Проверьте типы
mypy backend/
```

### 6. Закоммитьте изменения

```bash
git add .
git commit -m "feat: add new payment feature"
```

### 7. Отправьте в репозиторий

```bash
git push origin feature/your-feature-name
```

### 8. Откройте Pull Request

Перейдите на GitHub и создайте PR.

## 📝 Стандарты кода

### Стиль кода

- **Python**: PEP 8
- **Имена переменных**: `snake_case`
- **Имена классов**: `PascalCase`
- **Константы**: `UPPER_CASE`
- **Комментарии**: Русские или английские

### Пример

```python
from typing import Optional

logger = logging.getLogger("bot.payment")

    def __init__(self, wallet: str, secret: str):
        self.wallet = wallet
        self.secret = secret
    
    def process_payment(self, user_id: str, amount: int) -> bool:
        """
        Обрабатывает платёж пользователя.
        
        Args:
            user_id: ID пользователя в Telegram
            amount: Сумма платежа в рублях
            
        Returns:
            True если платёж успешен
        """
        logger.info(f"💳 Обработка платежа: user_id={user_id}, amount={amount}")
        # ... код ...
        return True
```

### Документирование

Используйте docstrings для всех функций и классов:

```python
def function_name(param1: str, param2: int) -> bool:
    """
    Краткое описание функции.
    
    Подробное описание (если нужно).
    
    Args:
        param1: Описание параметра
        param2: Описание параметра
        
    Returns:
        Описание возвращаемого значения
        
    Raises:
        ValueError: Описание исключения
    """
    pass
```

### Логирование

Используйте модуль `logging`:

```python
logger = logging.getLogger("bot.module_name")

# Уровни логирования
logger.debug("Отладочная информация")
logger.info("Информационное сообщение")
logger.warning("Предупреждение")
logger.error("Ошибка")
logger.critical("Критическая ошибка")
```

## 🔄 Pull Request процесс

### Требования к PR

1. **Один PR = одна функция/исправление**
2. **Тесты**: Добавьте тесты для новых функций
3. **Документация**: Обновите README.md если нужно
4. **Стиль**: Код должен соответствовать PEP 8

### Шаблон PR

```markdown
## Описание
Краткое описание изменений.

## Тип изменений
- [ ] 🐛 Bug fix (не ломает обратную совместимость)
- [ ] ✨ New feature (добавляет новую функцию)
- [ ] ⚠️ Breaking change (ломает обратную совместимость)
- [ ] 📝 Documentation (обновление документации)
- [ ] 🧪 Tests (добавление тестов)
- [ ] 🔧 Refactoring (рефакторинг без изменений функциональности)

## Чек-лист
- [ ] Код следует стандартам проекта
- [ ] Добавлены тесты (если применимо)
- [ ] Обновлена документация (если применимо)
- [ ] Все тесты проходят

## Тестирование
Как протестировать изменения:

1. ...
2. ...
3. ...

## Скриншоты (если применимо)
[Скриншоты]
```

### Ревью

1. **Code Review**: Один из мейнтейнеров проверит код
2. **CI/CD**: Автоматические тесты должны пройти
3. **Merge**: После одобрения PR будет merged
