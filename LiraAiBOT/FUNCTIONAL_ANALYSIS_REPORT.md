## 🎯 Текущий функционал LiraAI

### 1. Текстовый чат 💬

**Описание:** Общение с пользователем на русском языке через LLM

**Текущая реализация:**
- **Основные провайдеры:** Groq, Cerebras, OpenRouter
- **Модели:** Llama 3.3, Llama 4, Kimi K2, Solar, Trinity
- **Память:** История диалога (10 последних сообщений)
- **Режимы:** Текст, Голос, Фото, Генерация, Авто

**Требования к модели:**
- ✅ Поддержка русского языка
- ✅ Контекст минимум 8K токенов
- ✅ Скорость ответа < 2 секунд
- ✅ Качество ответов (адекватность, грамотность)

---

### 2. Анализ изображений (Vision) 📸

**Описание:** Распознавание и описание изображений, отправленных пользователем

**Текущая реализация:**
- **Файл:** `backend/api/telegram_vision.py`
- **Провайдеры:** Gemini Vision, Replicate LLaVA
- **Процесс:**
  1. Пользователь отправляет фото
  2. Бот скачивает изображение
  3. Отправляет в Vision API
  4. Получает описание
  5. Отвечает пользователю

**Требования к модели:**
- ✅ Vision API (мультимодальность)
- ✅ Поддержка русского языка в ответах
- ✅ Распознавание объектов, текста, сцен
- ✅ Скорость обработки < 10 секунд

**Пример промпта:**
```
Что изображено на этой фотографии? Опиши подробно на русском языке.
```

---

### 3. Распознавание голоса (STT) 🎤

**Описание:** Преобразование голосовых сообщений в текст

**Текущая реализация:**
- **Файл:** `backend/api/telegram_voice.py`
- **Движок:** `backend/voice/stt.py`
- **Библиотека:** SpeechRecognition (локальное распознавание)
- **Процесс:**
  1. Пользователь отправляет голосовое
  2. Бот скачивает аудио (OGG)
  3. Конвертирует в WAV (librosa)
  4. Распознает через SpeechRecognition
  5. Отправляет текст в LLM для ответа

**Требования к модели:**
- ✅ Распознавание русского языка (ru-RU)
- ✅ Поддержка формата OGG/WEBM (Telegram)
- ✅ Точность распознавания > 85%
- ✅ Скорость обработки < 5 секунд

**Зависимости:**
```python
pip install SpeechRecognition librosa soundfile
```

### 4. Синтез речи (TTS) 🔊

**Описание:** Преобразование текста в голосовые сообщения

**Текущая реализация:**
- **Файл:** `backend/voice/tts.py`
- **Движки:** ElevenLabs (основной), gTTS (fallback)
- **Процесс:**
  1. Пользователь просит озвучить ответ
  2. Бот генерирует аудио через ElevenLabs
  3. Отправляет как голосовое сообщение

**Требования к модели:**
- ✅ Поддержка русского языка
- ✅ Натуральное звучание
- ✅ Скорость генерации < 10 секунд

**API ключи:**
- `ELEVEN_API_KEY`
- `ELEVEN_VOICE_ID`
- `ELEVEN_MODEL_ID`

### 5. Генерация изображений 🎨

**Описание:** Создание изображений по текстовому описанию

**Требования к модели:**
- ✅ Генерация по текстовому промпту
- ✅ Качество изображения (минимум 512x512)
- ✅ Скорость генерации < 30 секунд
- ✅ Поддержка русского языка в промптах

**API ключи:**
- `POLZA_API_KEY`

### 3. Распознавание голоса - ✅ Groq Whisper (бесплатно!)

**Текущая реализация:** Локальный SpeechRecognition

**Рекомендация:** Интегрировать Groq Whisper API

**Преимущества:**
- ✅ Бесплатно (30 запросов/минуту)
- ✅ Высокая точность (>95%)
- ✅ Поддержка русского языка
- ✅ Не требует локальных библиотек

**Код интеграции:**
```python
# backend/voice/stt.py
from groq import Groq

class GroqSTT:
    def __init__(self, api_key):
        self.client = Groq(api_key=api_key)
    
    def speech_to_text(self, audio_path, language="ru"):
        with open(audio_path, "rb") as f:
            transcription = self.client.audio.transcriptions.create(
                model="whisper-large-v3",
                file=f,
                language=language
            )
        return transcription.text
```
